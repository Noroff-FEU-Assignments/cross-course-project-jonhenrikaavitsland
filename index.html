Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

To protect your website from web secuity attacks we advice you to implement the following practices:

  • Make sure that your hosting plaform is safe and secure.
  • Rewrite code to ensure that no markup is able to run hostile code.
  • Use web frameworks that include CSRF prevention mechanisms.
  • Make HTTP headers appropriate.
  • Sanitizing data where needed to prevent possible security breaches.
  • We highly encourage the use of stronger passwords and make sure that it is hard to crack your authorization and authentication, this is best achieved by utilizing open-standard authorization protocol or framework like Okla, OneLogin, etc.
  • To protect your users we advise you to implement automatic user timeout.
  • Utilize Continuous Deployment/Continuous Integration (CI/CD) when having developers work on your code. This will automatically check any changes made to the code and flag for rework code that else would be a security threat.
  • Make sure that all third-party applications that are being used are well-known, this ensures that the third-party application is trusted and tested. It might require that one take the time to research before any such application can be used.
  • Lastly and not to be forgotten, we urge you to configure your web server to use HTTPS and HTTP Strict Transport Security (HSTS). This is an encryption of the data sent between your server and your clients. This makes sure that data is not easily available to attackers.
